# 数学建模协会秘书处插件

## 项目简介

astrbot_plugin_mma_secretariat 是一个基于 AstrBot 框架开发的 QQ 机器人插件，专为数学建模协会秘书处设计。该插件实现了任务管理、部门协调、消息通知等核心功能，支持通过 QQ 群聊和 Web API 两种方式进行操作。通过该插件，秘书处可以高效地创建、分配、跟踪和管理协会各部门的工作任务，同时部门成员可以便捷地确认和反馈任务接收情况。

## 项目结构

该项目采用模块化设计，各组件职责明确，主要包含以下文件：

```
astrbot_plugin_mma_secretariat/
├── main.py                      # 插件入口文件，包含所有命令处理器
├── secretariat.py               # 核心业务逻辑类
├── secretariat_web.py           # Web API 处理模块
├── task_database.py             # 数据库操作模块
├── datetime_validator.py        # 时间格式验证模块
├── metadata.yaml                # 插件元信息配置
├── _conf_schema.json            # 配置文件 schema
├── README.md                    # 项目说明文档
├── LICENSE                      # 开许可证
├── .gitignore                   # Git 忽略配置
└── database/
    └── task_database.db         # SQLite 任务数据库
```

**核心模块说明：**

**main.py** 是插件的主入口文件，定义了 `MMASecretariatPlugin` 类，该类继承自 AstrBot 的 `Star` 基类。在该文件中，通过装饰器注册了所有 QQ 命令处理器，包括创建任务、发送任务、更新任务、删除任务、完结任务、查看任务等核心功能。同时，该文件还负责插件的初始化和销毁逻辑，包括启动内置的 Web 服务器用于接收外部请求。

**secretariat.py** 包含 `secretariat` 核心类，封装了所有业务逻辑方法。该类维护了管理员白名单、部门联系人字典、通知群组 ID 等配置信息，并提供了任务创建、查询、更新、删除、完结等静态方法。每个方法都包含权限检查机制，确保只有授权用户才能执行相应操作。

**secretariat_web.py** 实现了 Web API 功能，通过 aiohttp 框架构建异步 HTTP 服务器。该模块提供了与 QQ 命令等效的 Web 接口，允许外部系统通过 HTTP POST 请求调用插件功能。API 端点为 `/webhook`，支持 JSON 格式的数据交互。

**task_database.py** 负责数据库的 CRUD 操作，使用 SQLite 作为存储引擎。模块定义了 `Task` 数据类来映射任务实体，并提供了任务创建、查询、更新、删除、统计等静态方法。数据库采用单例模式维护连接，确保数据一致性。

**datetime_validator.py** 是一个时间格式验证和转换工具类，支持多种常见的时间格式输入。该模块可以将不同格式的时间字符串统一转换为 ISO 格式存储，同时提供格式示例查询功能，方便用户了解正确的输入格式。

## 功能特性

该插件提供了完整的任务管理功能，涵盖任务全生命周期的各个环节。

**任务管理功能** 支持创建新任务、查看任务列表、更新任务内容、删除任务、标记任务完结等操作。任务包含丰富的属性信息，包括任务ID、任务内容、截止时间、所属部门、任务状态、创建时间、更新时间、完成时间、回复状态等。任务状态分为“进行中”和“已完成”两种，系统会自动跟踪任务的执行进度。

**部门管理功能** 内置了五个标准部门：技术部、宣传部、组织部、学术部、外联部。每个部门可以绑定对应的联系人和群组，用于任务通知和消息推送。系统支持动态设置部门联系人和群组信息，无需修改代码即可适应组织架构变化。

**消息通知功能** 任务创建后可以通过“发送任务”命令通知相应部门。通知类型分为三种：通知（首次发布）、更新（任务内容变更）、完结（任务完成）。通知消息会 @相关部门的联系人，确保信息及时送达。

**回复确认功能** 部门成员收到任务通知后，可以回复“收到”或数字“1”来确认任务已接收。系统会自动标记该任务的回复状态，并向秘书处群组发送确认通知，实现双向沟通闭环。

**查询统计功能** 支持多种任务查询方式：查看全部任务、查看进行中任务、查看已完成任务、查看指定部门任务、查看指定ID任务详情。同时提供任务统计功能，显示总任务数、进行中数量、已完成数量、完成率以及各部门任务分布情况。

**权限管理功能** 采用白名单机制进行权限控制，分为根管理员和管理员两个级别。根管理员拥有最高权限，可以执行所有操作包括系统设置；管理员可以执行常规的任务管理操作。普通成员只能查看帮助信息和执行有限的查询操作。

**Web API 功能** 内置 HTTP 服务器，提供与 QQ 命令等效的 Web 接口。外部系统可以通过发送 HTTP POST 请求来触发任务管理操作，实现与其他系统的集成对接。

## 命令详解

插件提供丰富的命令系统，所有命令均支持中英文双语触发。

### 核心任务命令

**创建任务** 命令格式：`/创建任务 <内容> <截止时间> <部门>` 或 `/create_task <内容> <截止时间> <部门>`

示例：`/创建任务 完成活动策划 2026-12-31T23:59:00 组织部`

该命令用于创建新任务，需要提供任务的具体描述、截止时间以及接收任务的部门。创建成功后，系统会返回任务详情，包含自动分配的任务ID。时间格式支持多种输入方式，详见时间参数说明命令。部门名称必须使用完整名称，不可使用缩写。

**发送任务** 命令格式：`/发送任务 <任务ID> <类型>` 或 `/send_task <任务ID> <类型>`

示例：`/发送任务 1 通知`

发送任务通知到指定部门。任务类型分为三种：“通知”用于首次发布任务，“更新”用于通知任务内容已变更，“完结”用于通知任务已完成。发送成功后，相关部门的联系人和通知群组会收到消息。

**更新任务** 命令格式：`/更新任务 <任务ID> <内容> <截止时间> <部门>` 或 `/update_task <任务ID> <内容> <截止时间> <部门>`

示例：`/更新任务 1 / 2026-11-30T23:59:00 /`

更新任务信息。如果某个字段不需要修改，可以使用斜杠 `/` 占位。三个字段都使用斜杠占位时，命令会报错提示至少要提供一个要更新的字段。

**完结任务** 命令格式：`/完结任务 <任务ID>` 或 `/complete_task <任务ID>`

示例：`/完结任务 1`

将任务标记为已完成状态。完结后，任务状态会变更为“已完成”，并记录完成时间。

**删除任务** 命令格式：`/删除任务 <任务ID>` 或 `/delete_task <任务ID>`

示例：`/删除任务 1`

从数据库中删除指定任务。删除前会检查任务是否存在，删除后无法恢复。

**标记回复** 命令格式：`/标记回复 <任务ID>` 或 `/mark_received <任务ID>`

示例：`/标记回复 1`

手动标记指定任务已收到部门回复，与“收到”命令功能类似但不会向秘书处群组发送通知。

### 部门回复命令

**收到确认** 命令格式：`/收到 <任务ID>` 或 `/receive_task <任务ID>`

示例：`/收到 1`

部门成员确认收到任务通知。执行后会将任务标记为已收到回复，并向秘书处群组发送确认消息。

**快捷回复** 命令格式：`/1 <任务ID>` 或 `/receive_task_short <任务ID>`

示例：`/1 1`

收到确认的快捷方式，功能与“收到”命令完全相同，提供更便捷的操作体验。

### 查询命令

**查看任务** 命令格式：`/查看任务 <类型> [部门]` 或 `/view_tasks <类型> [部门]`

示例：

- `/查看任务 全部` 查看所有任务
- `/查看任务 进行中` 查看进行中的任务
- `/查看任务 已完成` 查看已完成的任务
- `/查看任务 部门 技术部` 查看技术部的任务
- `/查看任务 1` 查看ID为1的任务详情

该命令支持多种查询类型，可以按任务状态、部门或者直接使用任务ID进行查询。

**任务详情** 命令格式：`/任务详情 <任务ID>` 或 `/task_detail <任务ID>`

示例：`/任务详情 1`

查看指定任务的完整详细信息，包括所有字段内容。

**任务统计** 命令格式：`/任务统计` 或 `/task_stats`

示例：`/任务统计`

显示任务统计报表，包括总任务数、进行中数量、已完成数量、完成率以及各部门任务分布情况。

### 系统设置命令

**设置通知群聊** 命令格式：`/设置通知群聊` 或 `/set_notice_group`

在当前群聊中执行此命令，可将该群聊设置为任务通知的发送目标群组。仅根管理员可用。

**设置秘书处群聊** 命令格式：`/设置秘书处群聊` 或 `/set_secretariat_group`

在当前群聊中执行此命令，可将该群聊设置为秘书处内部工作群组。仅根管理员可用。

**设置部门联系** 命令格式：`/设置部门联系 <部门>` 或 `/set_department_contact <部门>`

在当前群聊或私聊中执行此命令，可将当前会话设置为指定部门的联系渠道。仅根管理员可用。

### 帮助命令

**部门参数说明** 命令格式：`/部门参数说明` 或 `/departments`

显示部门参数的正确使用方式，包括完整的部门列表和使用示例。

**时间参数说明** 命令格式：`/时间参数说明` 或 `/time_arg`

显示支持的时间格式列表、使用说明和示例，帮助用户正确输入时间参数。

**秘书处帮助** 命令格式：`/秘书处帮助` 或 `/help_secretariat`

显示完整的命令帮助信息，包括所有可用命令的格式说明和使用示例。

## 数据库说明

### 数据库结构

插件使用 SQLite 数据库存储任务数据，数据库文件位于 `database/task_database.db`。

**tasks 表结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 任务唯一标识符，自增主键 |
| content | TEXT | 任务内容描述 |
| deadline | TEXT | 截止时间，ISO格式字符串 |
| department | TEXT | 接收任务的部门名称 |
| status | TEXT | 任务状态，pending=进行中，completed=已完成 |
| created_time | TEXT | 任务创建时间 |
| updated_time | TEXT | 任务最后更新时间 |
| completion_time | TEXT | 任务完成时间，未完结时为“未完结” |
| response_received | BOOLEAN | 是否收到部门回复，0=未回复，1=已回复 |

### 数据操作

数据库操作封装在 `task_database` 类中，提供以下静态方法：

- `init_table()` 初始化数据库表结构
- `create_task(content, deadline, department)` 创建新任务
- `get_all_tasks()` 获取所有任务
- `get_pending_tasks()` 获取进行中的任务
- `get_completed_tasks()` 获取已完成的任务
- `get_tasks_by_department(department)` 获取指定部门的任务
- `get_task_by_id(task_id)` 根据ID获取任务
- `update_task(task_id, content, deadline, department)` 更新任务信息
- `mark_response_received(task_id)` 标记任务已收到回复
- `complete_task(task_id)` 完结任务
- `delete_task(task_id)` 删除任务
- `get_statistics()` 获取任务统计数据

所有数据库操作都包含异常处理机制，确保操作失败时能够记录日志并返回合适的错误信息。

## Web API 接口

插件内置 HTTP 服务器，默认监听 `0.0.0.0:8081` 端口，提供 Webhook 接口用于外部系统集成。

### 接口地址

```
POST http://<host>:<port>/webhook
```

### 请求格式

请求体为 JSON 格式，包含以下字段：

```json
{
    "messages": "<命令> <参数1> <参数2> ...",
    "user_id": "<用户QQ号>"
}
```

### 响应格式

响应体为 JSON 格式：

```json
{
    "success": true,
    "message": "操作结果信息"
}
```

或失败时：

```json
{
    "success": false,
    "message": "错误原因描述"
}
```

### 支持的命令

Web API 支持所有 QQ 命令，命令格式与 QQ 命令完全一致。例如：

```json
{
    "messages": "创建任务 完成策划案 2026-12-31T23:59:00 组织部",
    "user_id": "1727287294"
}
```

```json
{
    "messages": "查看任务 全部",
    "user_id": "1727287294"
}
```

### CORS 支持

API 已配置 CORS 中间件，允许跨域请求，支持 OPTIONS 预检请求。

## 配置文件

### metadata.yaml

插件元信息配置文件：

```yaml
name: astrbot_plugin_mma_secretariat
desc: 数学建模协会小数秘书处模块
version: 1.0.0
author: dreamDust
repo: https://github.com/mmaupc/astrbot_plugin_mma_secretariat.git
```

### _conf_schema.json

配置项 Schema 定义文件，定义了以下配置项：

- `notice_group_id` 通知群组ID，用于发送任务通知
- `secretariat_group_id` 秘书处群组ID，用于内部沟通
- `department_contacts` 部门联系人字典，键为部门名称，值为联系人QQ号或群号
- `web_host` Web服务器监听地址，默认 0.0.0.0
- `web_port` Web服务器监听端口，默认 8081

## 权限说明

### 根管理员

根管理员拥有最高权限，可以执行所有操作，包括：

- 所有任务管理操作
- 系统设置命令
- 配置文件修改

根管理员 QQ 号在 `secretariat.py` 中的 `ROOT_WHITELIST` 列表定义，默认包含 `1727287294`。

### 管理员

管理员可以执行常规的任务管理操作，但不能修改系统设置。管理员 QQ 号在 `ADMIN_WHITELIST` 列表定义，默认与根管理员相同。

### 普通成员

普通成员只能查看帮助信息和执行有限的查询操作，无法创建、修改或删除任务。

## 注意事项

1. 首次使用前，需要通过设置命令配置通知群组、秘书处群组和部门联系人信息。
2. 时间格式推荐使用 ISO 格式 `YYYY-MM-DDTHH:MM:SS`，例如 `2026-12-31T23:59:00`。
3. 部门名称必须使用完整名称，如“组织部”而非“组织”。
4. Web API 默认端口为 8081，如需更改可在配置文件中修改。
5. 数据库文件会自动创建在 `database/task_database.db`，无需手动初始化。
